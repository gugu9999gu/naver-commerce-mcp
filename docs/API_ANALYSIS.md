# 네이버 커머스 API 상세 분석과 MCP 구현 범위

기준 문서는 **커머스 API 2.80.0(2026-06-10)** 이다. 네이버 문서의 `current` 경로는 이후 바뀔 수 있으므로 운영 반영 전에 변경 이력을 다시 확인한다.

## 1. 시스템 구조

커머스 API는 스마트스토어 기능을 서버 간 HTTP API로 제공한다.

```text
https://api.commerce.naver.com/external
```

일반 요청·응답은 JSON/UTF-8이며 날짜와 시간은 ISO 8601, KST 오프셋 `+09:00`을 사용한다. 장애 분석과 네이버 문의를 위해 응답 헤더 `GNCP-GW-Trace-ID`와 `GNCP-GW-HttpClient-ResponseTime`을 보존한다.

## 2. 인증 구현

OAuth 2.0 Client Credentials를 사용한다. 토큰 요청에서 애플리케이션 시크릿 자체를 전송하지 않고 다음 전자서명을 만든다.

```text
password = client_id + "_" + timestamp_ms
bcrypt_hash = bcrypt(password, salt=client_secret)
client_secret_sign = Base64(bcrypt_hash)
```

토큰 엔드포인트는 `POST /v1/oauth2/token`이다. `SELF`는 애플리케이션 자체 리소스, `SELLER`는 연결 판매자 리소스이며 SELLER에는 `account_id`가 필요하다.

MCP 구현은 토큰을 메모리에 캐시하고 만료 전에 갱신한다. API 응답이 `401`이고 오류 코드가 `GW.AUTHN`이면 토큰을 지우고 한 번만 재발급·재시도한다.

## 3. 실제 업무 도메인

### 상품

원상품은 판매가·옵션·재고·배송·상세 정보 등 핵심 상품 정보를 담고, 채널상품은 스마트스토어 등 채널별 노출 정보를 가진다. 상품 등록 전에 카테고리, 속성, 원산지, 상품정보제공고시, 옵션, 브랜드·제조사를 조회해야 한다.

상품 이미지는 `POST /v1/product-images/upload`의 multipart `imageFiles`로 업로드한다. 이 MCP는 로컬 허용 디렉터리의 이미지 1~10개를 읽어 업로드하고, 반환 URL을 상품 등록 요청에 사용할 수 있게 한다.

등록과 수정 요청은 카테고리별 필수 필드가 매우 많고 자주 확장되므로 MCP가 임의로 단순화하지 않고 공식 JSON 요청 객체를 그대로 입력받는다. 대신 조회·이미지 업로드·등록·수정·삭제·상태·재고 작업을 별도 명명 도구로 제공한다.

### 주문과 클레임

`orderId`와 `productOrderId`는 다르며 발주·발송·취소·반품·교환은 주로 상품주문 단위로 처리된다.

안전한 주문 처리 순서:

1. 변경 상품 주문 목록 조회
2. 상품주문 ID로 최신 상세 상태 조회
3. 현재 상태가 허용하는 전이인지 확인
4. 발주 확인 또는 발송·클레임 요청 실행
5. 실행 결과와 Trace ID 기록
6. 상세 정보를 다시 읽어 최종 상태 확인

발주 확인과 발송은 한 요청에 최대 30건으로 제한한다. 상세 일괄 조회는 최대 300건이다.

### 문의

상품 Q&A와 네이버페이 고객 문의는 서로 다른 API다. 이 MCP는 두 목록 조회와 각각의 답변 등록·수정을 구분해 제공한다. 답변은 고객에게 노출되는 실제 데이터이므로 쓰기 보호를 적용한다.

### 정산

건별·일별 부가세, 건별·일별 정산, 수수료 상세 조회를 제공한다. 날짜·페이지 조건은 최신 공식 문서 형식을 그대로 전달한다.

### N배송

SKU 목록, SKU 상세, SKU 연결상품 조회를 지원한다. 네이버 SKU ID와 판매자 상품·옵션 ID는 별도 키로 저장하는 것이 안전하다.

## 4. 호출 제한과 재시도

- `429`, `502`, `503`, `504`: GET과 명시된 조회용 POST만 제한적으로 지수 백오프 재시도
- `401 + GW.AUTHN`: 토큰 재발급 후 한 번 재시도
- `400`: 스키마나 코드 오류이므로 자동 재시도하지 않음
- `403`: API 권한, 판매자 상태, 약관, 허용 IP 등을 확인
- 쓰기 요청: 중복 처리 위험 때문에 자동 재시도하지 않음

## 5. MCP 보안 모델

- HTTPS 기본 호스트 고정
- `/v1/`, `/v2/` 상대 경로만 허용
- 절대 URL, 역슬래시, 경로 이동 차단
- 자격 증명과 토큰을 도구 결과에 노출하지 않음
- 쓰기 기본 차단
- `NAVER_COMMERCE_ALLOW_MUTATIONS=true`와 `confirm=EXECUTE` 이중 조건
- 이미지 realpath 확인, 허용 루트 격리, 크기·확장자·개수 제한
- stdio 사용, stdout은 MCP JSON-RPC 전용

## 6. 직접 제공하는 도구 범위

- 연결 검증
- 판매자 계정·채널·주소록
- 상품 검색·상세·등록 지원 데이터
- 이미지 업로드
- 상품 등록·수정·삭제·판매 상태·옵션 재고·멀티 수정
- 주문 증분·기간·상세 조회
- 발주 확인·발송·발송 지연
- 취소·반품·교환 상태 처리
- 상품 문의·고객 문의 조회와 답변
- 부가세·정산·수수료 조회
- N배송 SKU 조회
- 공식 JSON 엔드포인트용 읽기/쓰기 확장 도구

## 7. 검증 범위

자동 테스트는 서명 생성, 경로 방어, 조회 분류, 이미지 파일 격리, 인증 만료 복구, 재시도 토큰 재사용을 검증한다. MCP stdio 연결로 도구 34개, 리소스 2개, 프롬프트 1개 노출을 확인한다.

실제 판매자 계정에 대한 호출 성공 여부는 사용자의 `client_id`, `client_secret`, 애플리케이션 권한, 판매자 상태에 의존한다. 자격 증명 없이 라이브 상품·주문을 임의로 변경하는 테스트는 수행하지 않는다.
